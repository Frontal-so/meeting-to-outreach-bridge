One booking. The whole account starts hearing from you.

A Calendly booking adds the person who booked to a Lemlist connection campaign, then hands their company's domain to Clay. Clay finds the director and VP level people in growth, marketing, and sales there, so your team can start reaching out before the call ever happens.

What actually happens

The moment someone books

This is the real payload Calendly sends, and the two things the app does with it. Nothing abstract, this is the whole job.

POST /webhook/booking
{
  "event": "invitee.created",
  "payload": {
    "email": "jane@acme.co",
    "first_name": "Jane",
    "questions_and_answers": [
      { "question": "LinkedIn profile URL", "answer": "linkedin.com/in/janedoe" },
      { "question": "Company website", "answer": "acme.co" }
    ]
  }
}

→ Lemlist: Jane added to the connection campaign
→ Clay: acme.co sent for stakeholder enrichment

What you get

One small Flask app, built to be forked and pointed at your own accounts.

  • A working webhook receiver, about 140 lines, that parses the real Calendly payload shape, including custom questions, not flattened placeholder fields.
  • Signature verification using Calendly's actual signing key, so random requests can't hit your endpoint.
  • A one-command local test with a real example payload and curl command.
  • Step-by-step Railway deploy instructions, plus a `.env.example` for every key you need.
  • What to build inside Clay to turn a bare domain into a list of the right people.
  • MIT licensed. Fork it, swap in your keys, ship it.

Clay setup

What to build inside Clay

The app's job ends when Clay receives a domain. Everything from here happens inside your Clay workspace.

  1. Webhook table. One row per booking. domain is the column everything else builds off.
  2. Find people at the company. Run a people-search enrichment on domain. This expands into one row per person found.
  3. Filter to the right level. Director or VP, in growth, marketing, or sales. Drop everyone else.
  4. Confirm each LinkedIn URL. Most people-search enrichments return it directly.
  5. Send to Lemlist, split across senders. Point the filtered rows at more than one campaign, one per teammate sending connection requests, so it's not just you showing up in every inbox.

Before you start

What you need first

  • A Calendly account where you can add webhook subscriptions and custom questions
  • A Lemlist account, a connection campaign, and its API key
  • A Clay workspace with a webhook-triggered table
  • Somewhere to run the app continuously, the repo is set up for Railway
